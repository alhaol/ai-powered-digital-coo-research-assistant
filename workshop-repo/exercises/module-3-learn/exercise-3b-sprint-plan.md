# Exercise 3B — Generate a Pivot Plan (2-Week Sprint)

**Module:** 3 — LEARN · **Time:** ~6 min · **Format:** Hands-On + Independent Build

> A decision without a plan is just a wish. Turn the recommendation into 14 days
> of concrete, budgeted action.

## Goal
Produce a 2-week sprint plan for the next iteration: updated hypothesis, 5
prioritized tasks, success metrics, a bootstrap budget, and risks.

## Do This
```bash
claude "Based on the pivot to 'ingredient transparency' positioning,
create a 2-week sprint plan for my sustainable skincare startup.
Include:
- Updated hypothesis
- 5 prioritized tasks with owners (assume I'm a solo founder)
- Success metrics for each task
- Estimated budget (bootstrapped, under \$500)
- Key risks and mitigations
Format as a markdown document with a Gantt-style text timeline."
```

## What Good Looks Like
- Tasks are *measurable* ("50 pre-orders"), not vague ("build brand")
- Budget lines actually sum to under $500
- A single clear "did this sprint win?" metric

See `sample-outputs/sprint_plan_example.md`.

## 🔧 Then — Build the Command Yourself (Independent)
➡️ Open `exercise-3b-cmd-build-command.md`. Build `.claude/commands/sprint-plan.md`
independently.

## The Loop Is Now Complete
`hypothesis` → `competitive-analysis` + `analyze-survey` → `pivot-decision` →
`sprint-plan`. That's the entire Build-Measure-Learn loop, codified as commands
that travel with your repo in Git.
